CSC110 Syllabus

Course Material

Primary Resources

  • Video Lessons: Professor Blake and I are creating video lessons to cover most of the topics in this class. These are intended to introduce topics and prepare you for in-class activities.
  • Class Activities/Examples: We will have practice activities during class, coding demonstrations, and reviews in which I show solutions to previous activities. Any examples that I create during class will be posted online after the meeting.
  • Notes: I'm in the process of revising my notes for this class, and they will be updated frequently during the semester. I use the notes for reading assignments after we've covered a topic in class.
  • Textbooks: You do not need to purchase any textbooks for this class. I try to cover everything in my notes, but you may want to use a textbook for further reading or to see a different explanation of a topic. I've linked two free textbooks from libretexts.org below. I may also use these for reading assignments if I'm behind on my notes:
    • Programming and Computation Fundamentals, by Busbee and Braunschweig
    • Computer Science from the Bottom Up, by Wienand

Software

  • Text Editors: You need a text editor to edit code (Microsoft Word and Google Docs are not text editors). I've listed a few suggestions below, but there are many other options out there. It does not matter to me which editor you use as long as you learn how to use it.
    • VS Code: I believe this is the most popular editor currently available. I recommend using it unless you have a good reason to go with another editor. One caveat: I do not personally use VS Code, and I cannot provide much technical support for it. However, due to its popularity you should be able to find plenty of resources online.
    • Notepad++: I like Notepad++, but it's Windows only and I haven't used it in years. It still appears to be receiving regular updates, so it should be a reasonable choice if you're using Windows and don't like VS Code.
    • Neovim: This is the editor I use. It's harder to learn than the other options in this list, but it's powerful and highly configurable. If you don't mind spending time learning vim motions and various keyboard shortcuts, then it might be worth giving neovim a try. You could also consider vim and emacs, both of which have similar pros and cons to neovim. However, I think most people will be happier with VS Code, so I don't recommend these options unless one appeals to you and you're fine with the learning curve.
  • Java: This is the programming language we're using. You'll set up Java during the first lab, and I'll post directions on Blackboard in case you need to reinstall it in the future.
  • Git Bash: We'll use a terminal regularly. I don't like the default terminal programs on Windows (cmd and Powershell), and some of the scripts and commands I give you will not work in either of them. Git for Windows comes with a decent bash emulator, and I highly recommend it over cmd and Powershell if you use Windows. If you're using a Mac or Linux computer, then the default terminal program (usually bash or zsh) is fine.
  • diagrams.net: This is a web app for creating flowcharts and other diagrams. It can also integrate with Google Drive and allow multiple people to edit the same diagram, but some students have reported issues with lag while doing this.

Other Resources

The Java API is a useful reference. It may be intimidating right now, but the more you learn about Java and programming in general, the more easily you'll be able to navigate the API and use it to look up information.

Finally, asking good questions is one of the most useful skills you can have as a student. In my experience, most people (myself included) don't ask questions as often as they should, and it can be difficult to ask a question that will quickly lead to a helpful answer. I think the blog posts below provide excellent advice for asking and answering questions:
